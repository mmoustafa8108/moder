# Contributions

contributions are more than welcome, always feel free to pull request or create new issues or share discussions, as they push the project forward!

but when working on a new feature or changing something in the design/architecture of the project, always ensure you discussed it either in discussions or issues section and always ensure it got approval from experienced people to increase the possibility of accepting your new design/architecture/feature

when fixing bugs or working on code that doesn't change the design/architecture of the project, always ensure following the [syntax convention rules](./syntax_conventions.md) as they're an important anchor when reviewing pull request, and not following them will indicate lack of consistency and careness about the code, because one of the goals of moder is to be easy to change as easy to use.

also, be careful with git commits. we don't want a messy git repo full of commits for each new line yet we don't need it full of 1000 new lines commits,
clean commits indicate clean workflow.

if you changed any part of the code that makes any part of the docs outdated, you should update it yourself and don't wait for someone to update it after you, this isn't to bother you but it's to ensure you REALLY understand the code that you can explain it

*(no need to note that unconcious usage of AI is the fastest way to get your pull request rejected, and that spamming is the fastest way to get blocked at all).
